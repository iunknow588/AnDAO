# AnDaoWallet H5 —— 投资者简报

## 1. 项目概览
- **形态**：纯前端 PWA 智能合约钱包，无自建后端，部署即用。
- **内核**：基于 Kernel / ERC-4337（账户抽象），支持 UserOperation、Bundler、Paymaster。
- **三路径入门**：A 极简零门槛（无 EOA/Gas）、B 标准兼容（有 EOA，可自付或申请赞助）、C 赞助商（代付+审核+渠道）。
- **关键场景**：支持两阶段提交 / 比特承诺（Commit/Reveal）通用能力——一次授权，二阶段自动发送，无需二次签名。

## 2. 面向市场的价值
- **拉新与转化**：路径A零门槛 + 赞助商代付，降低 Web3 初次体验流失。
- **留存与活跃**：路径B保留 EOA 兼容性，用户可自付或申请赞助，降低长期摩擦。
- **生态运营**：路径C为发行方/社区运营者提供审核、代付、渠道与规则，形成正向网络效应。
- **开发者友好**：标准 EIP-1193 / EIP-6963 Provider，可编程交易（批量、条件、延迟、两阶段提交），易于嵌入 DApp。

## 3. 产品亮点
- **完全无后端**：不落地用户数据，降低合规与运维负担。
- **安全默认开启**：私钥/助记词本地加密存储，默认不导出明文；显式授权、最小权限。
- **自动化两阶段提交**：Commit→Reveal 自动发送（用户首阶段已授权），适用于投票、抽签、承诺式交互。
- **批量与插件化**：一次签名多调用，支持插件（比特承诺、延迟交易等）扩展复杂交互。
- **可插拔存储**：默认 IPFS，链上索引保存存储标识符；可接入自定义存储。
- **跨平台即开即用**：PWA，移动/桌面浏览器均可使用，无需插件或扩展。

## 4. 目标用户与场景
- **Web3 新手**：无需 EOA/Gas 即可获得智能合约账户（路径A）。
- **成熟用户/交易者**：兼容 EOA，享受智能账户高级能力（路径B）。
- **运营方 / 赞助商**：代付 Gas、审核、渠道与限额配置（路径C）。
- **DApp 开发者**：需要可编程钱包、批量/条件交易、两阶段提交能力的应用方。

## 5. 技术与架构要点
- **账户抽象**：Kernel 智能账户 + ERC-4337 EntryPoint + Bundler/Paymaster。
- **Provider 标准**：EIP-1193、EIP-6963，覆盖连接/签名/交易确认队列。
- **安全与合规**：本地加密、会话锁定、权限最小化；日志与错误分级，支持导出排障。
- **性能与可用性**：代码分割、缓存、超时与重试；外部服务故障时有清晰降级提示。

## 6. 商业潜力
- **LTV 提升**：赞助商模式降低获客成本，增强新用户进入与留存。
- **生态扩展**：插件化与两阶段提交可服务于投票、公证、抽签、条件交易等多种链上业务。
- **低运维成本**：纯前端形态降低基础设施投入，可快速克隆/多环境部署。

## 7. 路线与里程碑（示例）
- **MVP**：三路径创建、基础资产与交易、Provider、赞助商申请/审核、IPFS 存储落地。
- **增强**：批量与条件交易插件、自动化两阶段提交 UI、Paymaster 优化、更多链支持。
- **生态**：赞助商渠道与分析面板、DApp 集成模板、运营级风控规则。

## 8. 联系与演示
- **演示形态**：PWA，静态部署即可运行；需配置外部 RPC/Bundler/Paymaster/IPFS。
- **适配场景**：NFT/DeFi/DAO/DID 等需要低门槛与可编程交易的链上应用。
